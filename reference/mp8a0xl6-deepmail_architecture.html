06 · Security Model
Defense in Depth — Every Layer

File Quarantine

Uploaded files UUID-renamed on arrival. Stored 0o400 (read-only, no execute). Isolated S3 bucket. No process can write post-upload. Path traversal validated via canonical path check.

Sandbox Isolation

URL sandbox: gVisor (user-space kernel intercepts all syscalls). Dynamic sandbox: Firecracker microVM (hardware virtualization boundary). Ephemeral — destroy after every job, no cross-job state, no host network.

Auth — No Third Party

Email + password + OTP. No OAuth, no SSO. Argon2id hashing (PHC winner). RS256 JWT (asymmetric — private signs, public verifies). OTP: 8-char alphanumeric, hashed in DB, 10min TTL, 5 attempt max.

Brute Force Protection

5 failed logins → 15 min IP lockout (exponential on repeat). login_attempts table tracks all attempts. Redis token bucket for rate limiting at gateway (per-IP, per-tenant). ip_lockouts with lockout_count.

Memory Safety

All file-processing services in Rust. No buffer overflows, no use-after-free, no data races by construction. Zero unsafe blocks in production paths. Processes malicious attachments without C-extension risk.

SQL Injection — Zero Surface

SQLx query! macro compiles all SQL at build time. No string-interpolated queries anywhere. Parameterized at the type level. ORM-free. CIDR matching via GIST index (not string concat).

mTLS — Service Mesh

Linkerd injects mTLS sidecar to every pod automatically. All gRPC between services is mutually authenticated and encrypted. No unauthenticated internal calls even within the cluster.

Secret Management

HashiCorp Vault. Dynamic DB credentials rotated every 1hr. API keys (AbuseIPDB, VT, etc.) in Vault KV. PKI for mTLS certs auto-renewed. No secrets in env files, config.toml, or Docker images.

Multi-layer File Validation

Step 1: extension check (.eml/.msg). Step 2: file size limit. Step 3: magic bytes (libmagic). Step 4: MIME type validation. Step 5: canonical path (no traversal). All steps logged to file_validations table.

Tenant Isolation

tenant_id on every table. All queries filter by tenant_id. NATS subjects include tenant routing. WS connections verified against JWT tenant claim. Graph service enforces per-tenant node visibility in Cypher queries.

API Key Security

Format: dm_live_{32 random chars}. Only prefix (first 12 chars) shown to user in UI. Full key SHA-256 hashed in DB. Never stored in plaintext after creation. Scope list enforced per key.

Idempotency Guards

Billing: UUIDv5(nats_msg_id + event_type) → UNIQUE constraint prevents double-billing on NATS replay. Razorpay webhooks: event_id UNIQUE column. OTP: single-use boolean column. Refresh token: single-use rotation.

07 · Infrastructure
Kubernetes Deployment Architecture
Kubernetes Resource Model
  • 1 Deployment per service — independent rollout, independent scaling
  • HPA on sandbox services — scale sandbox-url/file/dynamic 1→20 pods based on NATS consumer lag
  • HPA on scoring/ml — scale on CPU (ML inference is CPU-bound)
  • PodDisruptionBudget — min 1 replica available during rolling update for all services
  • ResourceQuota per namespace — sandbox namespace has strict CPU/memory limits
  • NetworkPolicy — sandbox pods have no egress to cluster network (only NATS and S3)
  • ServiceAccount per service — RBAC for Vault secret access

09 · Auth Flow
Registration & Login — No Third Party
Registration flow
1

Submit: username + email + password

All three required. Email CITEXT unique. Username unique. Password min 12 chars, entropy checked.

2

Argon2id hash password

time_cost=3, memory=65536KB, parallelism=4. Never stored plaintext.

3

Generate 8-char OTP

rand::thread_rng() → alphanumeric. Argon2id hash stored. Plaintext sent via SMTP only.

4

User enters OTP
